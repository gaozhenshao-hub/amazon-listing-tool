# 产品落地流程优化方案

> 基于现有代码结构的全面审计，本文档以表格形式梳理产品落地6大阶段的现状、问题与优化方向，核心目标是将"纯AI文本输出"升级为"结构化表格 + 人工可编辑确认"模式。

---

## 一、产品落地流程总览

产品落地流程共6个阶段，在评分立项通过后依次解锁。当前各阶段的交互模式和数据结构差异较大，需要统一为"AI生成 → 表格化展示 → 人工编辑 → 确认锁定"的标准范式。

| 阶段 | Tab名称 | 当前状态 | 核心问题 | 优化优先级 |
|------|---------|---------|---------|-----------|
| 1 | 评分立项 | 已有6维度评分卡片 + AI推理文本 | 评分维度不可编辑，AI推理为纯Markdown文本 | ★★★ |
| 2 | 产品画像 | 8子模块，支持AI生成/编辑/确认 | 编辑器为纯文本textarea，AI输出为JSON原文 | ★★★★ |
| 3 | BOM表 | 表格展示，AI建议BOM | 表格只读，不支持行内编辑和手动新增 | ★★★★★ |
| 4 | 说明书 | 9章节双语，支持HTML/PDF导出 | 章节内容不可逐章编辑确认 | ★★★ |
| 5 | 测试报告 | 8类测试项表格，支持状态切换 | 缺少实际结果填写和备注编辑 | ★★★★ |
| 6 | 利润计算 | 参数输入 + 批量模拟表格 | 模拟结果不可保存，缺少历史对比 | ★★★ |

## 二、各阶段详细优化框架

### 阶段1：评分立项

**现状**：AI一次性输出6维度评分（0-20分/维度），总分100分制。评分结果以数字卡片展示，AI推理以Streamdown纯文本渲染。用户只能"确认立项"或"撤销立项"，无法调整单项评分。

| 维度 | 现有展示 | 问题 | 优化方向 |
|------|---------|------|---------|
| 市场容量 | 数字卡片（只读） | 用户无法根据实际调研修正 | 改为可编辑数字输入 + 滑块，范围0-20 |
| 竞争强度 | 数字卡片（只读） | 同上 | 同上 |
| 利润空间 | 数字卡片（只读） | 同上 | 同上 |
| 差异化 | 数字卡片（只读） | 同上 | 同上 |
| 入场机会 | 数字卡片（只读） | 同上 | 同上 |
| 风险 | 数字卡片（只读） | 同上 | 同上 |
| AI推理 | Streamdown纯文本 | 不可编辑，无法补充人工判断 | 拆分为6维度独立推理卡片，每卡片可编辑 |
| 总分 | 自动计算（只读） | 修改单项后需自动重算 | 实时计算，低于阈值自动标红警告 |

**优化后交互流程**：

```
AI评分 → 6维度评分表格（每行：维度名 | AI评分 | 人工调整 | AI推理 | 人工备注）
       → 实时总分计算
       → 确认立项 / 驳回
```

### 阶段2：产品画像（8子模块）

**现状**：8个子模块各自独立，AI输出JSON格式建议，用户通过textarea编辑JSON原文后确认锁定。编辑体验差，非技术用户难以操作JSON。

| 子模块 | AI输出格式 | 现有编辑方式 | 问题 | 优化方向 |
|--------|-----------|------------|------|---------|
| 外观设计 | JSON: colors[], materialSuggestion... | textarea编辑JSON | 用户需手动编辑JSON | 改为结构化表格：颜色表 + 材质选择 + 表面处理 |
| 功能提升 | JSON: mainFunctions[], upgrades[] | textarea编辑JSON | 同上 | 改为功能列表表格，每行可编辑优先级和描述 |
| 产品成本 | JSON: breakdown[], targetRetailPrice... | textarea编辑JSON | 同上 | 改为成本明细表格，支持行内编辑金额 |
| 包装设计 | JSON: dimensions{}, weight... | textarea编辑JSON | 同上 | 改为尺寸参数表单 + 结构选择下拉 |
| 包装外观 | JSON: designStyle, colorScheme... | textarea编辑JSON | 同上 | 改为设计风格选择 + 配色方案卡片 |
| 用户画像 | JSON: demographics{}, painPoints[] | textarea编辑JSON | 同上 | 改为人口统计表格 + 痛点列表 |
| 使用场景 | JSON: scenarios[], seasonalTrends | textarea编辑JSON | 同上 | 改为场景卡片列表，每卡片可编辑 |
| 产品地图 | JSON: positioning{}, competitors[] | textarea编辑JSON | 同上 | 改为竞品对比表格 + 定位参数表单 |

### 阶段3：BOM表

**现状**：AI建议BOM后以只读表格展示（部件名称、材质、工艺、数量、单价、备注），不支持行内编辑、新增行、删除行。

| 功能点 | 现有状态 | 问题 | 优化方向 |
|--------|---------|------|---------|
| BOM列表 | 只读表格 | 无法修改AI建议的材质/价格 | 改为行内可编辑表格 |
| 新增部件 | 不支持 | 无法手动补充遗漏部件 | 增加"新增行"按钮 |
| 删除部件 | 不支持 | 无法移除不需要的部件 | 增加行级删除操作 |
| 供应商关联 | 不存在 | BOM与供应商库未打通 | 增加供应商选择列 |
| 模具费分摊 | 利润计算器中手动输入 | 与BOM数据未联动 | BOM中增加模具费列，自动汇总 |
| 成本汇总 | 不存在 | 无法直观看到总成本 | 表格底部增加成本汇总行 |
| 确认锁定 | 不存在 | BOM无确认状态 | 增加整表确认/解锁机制 |

## 三、统一交互范式

所有阶段应遵循统一的"AI生成 → 表格展示 → 人工编辑 → 确认锁定"交互范式：

| 步骤 | 交互元素 | 说明 |
|------|---------|------|
| 1. AI生成 | [AI生成] 按钮 | 调用AI生成初始数据，填充表格 |
| 2. 表格展示 | 结构化表格 | AI结果以表格形式展示，非JSON/Markdown文本 |
| 3. 人工编辑 | 行内编辑 / 弹窗编辑 | 每个单元格可点击编辑，支持新增/删除行 |
| 4. 确认锁定 | [确认锁定] 按钮 | 锁定当前版本，进入下一阶段 |
| 5. 解锁重做 | [解锁] 按钮 | 解锁后可重新编辑或重新AI生成 |

**状态机**：

```
空白 → [AI生成] → AI草稿 → [人工编辑] → 编辑中 → [确认锁定] → 已锁定
                                                        ↓
                                                   [解锁] → 编辑中
```

## 六、总结

本次优化的核心理念是**"AI是助手，人是决策者"**。每个阶段的AI输出都应该是结构化的、可编辑的表格数据，而非不可操作的纯文本。用户在AI建议的基础上进行调整、补充和确认，最终形成经过人工审核的高质量产品落地方案。
