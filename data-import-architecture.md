# 数据导入中心 — 架构方案（替代领星API）

## 一、改造思路

**核心变化**：从"系统自动调领星API拉数据" → "用户从领星ERP导出Excel → 上传到本系统 → 系统解析入库"

**优势**：
- 彻底消除API鉴权、签名、频率限制、代理等问题
- 数据更可控，用户可以选择性导入
- 领星ERP本身的导出功能非常完善，覆盖所有数据类型

**用户操作流程**：
1. 在领星ERP中导出对应报表（Excel/CSV）
2. 在本系统"数据导入中心"选择数据类型
3. 上传文件 → 系统自动识别列名 → 预览数据 → 确认导入

### 上传交互流程
```
1. 用户点击某数据类型的"上传"按钮
2. 弹出上传Dialog:
   - 顶部: 数据类型说明 + 领星导出路径提示
   - 中间: 拖拽上传区域（支持xlsx/csv）
   - 上传后: 自动解析 → 显示列名映射预览
   - 预览表格: 前5行数据预览
   - 底部: "确认导入" / "取消"
3. 导入后显示结果: 新增X条，更新X条，跳过X条
```

## 四、后端架构

### 数据库新增表
```sql
-- 导入记录表
CREATE TABLE data_imports (
  id INT AUTO_INCREMENT PRIMARY KEY,
  userId VARCHAR(100) NOT NULL,
  dataType VARCHAR(50) NOT NULL,        -- 'profit_report', 'fba_inventory', etc.
  fileName VARCHAR(255) NOT NULL,
  fileUrl VARCHAR(500),                  -- S3 URL
  rowCount INT DEFAULT 0,
  importedCount INT DEFAULT 0,
  updatedCount INT DEFAULT 0,
  skippedCount INT DEFAULT 0,
  errorCount INT DEFAULT 0,
  status VARCHAR(20) DEFAULT 'pending',  -- pending, processing, completed, failed
  errorMessage TEXT,
  dateRange VARCHAR(100),                -- 数据覆盖的日期范围
  createdAt BIGINT NOT NULL
);
```

### 解析器架构
```
server/
  dataImport/
    index.ts              -- 路由入口
    parsers/
      profitReport.ts     -- 利润报表解析器
      fbaInventory.ts     -- FBA库存解析器
      productList.ts      -- 产品列表解析器
      asin360.ts          -- ASIN360解析器
      spAdReport.ts       -- SP广告报表解析器
      searchTermReport.ts -- 搜索词报告解析器
      reviewData.ts       -- 评论数据解析器
      ...
    columnMapper.ts       -- 智能列名映射（中英文列名 → 系统字段）
    validator.ts          -- 数据校验
```

## 五、改造影响范围

### 需要改造的模块

| 模块 | 当前方式 | 改造后 | 复杂度 |
|------|---------|--------|--------|
| 产品总览-产品同步 | API拉取产品列表 | 上传产品列表Excel | 中 |
| 产品总览-周度数据 | API拉取利润+广告+库存 | 上传利润报表+广告报表 | 高 |
| 广告分析-全部Tab | API拉取广告数据 | 上传广告报表Excel | 高 |
| 库存管理 | API拉取FBA/AWD/本地仓 | 上传库存Excel | 中 |
| 售后服务 | API拉取评论/退货/邮件 | 上传评论+退货Excel | 中 |
| 物流发货 | API拉取入库计划 | 上传入库计划Excel | 低 |
| 竞品监控 | API拉取竞品数据 | 上传竞品监控Excel | 低 |
| 补货引擎 | API拉取库存+预测 | 上传库存+补货建议Excel | 中 |
| 定时任务(cronJobs) | 自动API同步 | 移除自动同步，改为提醒用户上传 | 低 |

### 可以保留的部分
- **店铺列表**: 改为用户手动配置（已有系统设置页面）
- **AI分析逻辑**: 完全不变，只是数据来源从API改为数据库
- **前端展示**: 大部分不变，只是数据加载方式改变
