## 核心规则

### 1. 标题结构改为两段式
- **基础信息层（Title）**：长度建议 `<= 75` 字符（含空格）
- 结构公式：`品牌名 + 核心品词（大词/流量词） + 最关键属性（型号/尺寸/核心参数）`
- 高流量核心词要尽量前置
- 不要混入营销词汇或冗长场景词

### 2. 亮点层（Item Highlights）
- 长度建议 `<= 125` 字符（含空格）
- 结构公式：`核心功能 + 竞争亮点/差异化卖点 + 使用场景/适用人群`
- 应承接标题中的高转化长尾词、场景词和差异化属性

### 3. 文案优化五个维度
1. **场景化描述**：围绕买家使用场景组织文案
2. **问题导向型文案**：直接回答用户问题
3. **关键词密度与语义关联**：构建核心词、相关词、长尾词、场景词的分层布局
4. **情感化与信任背书**：如质保、认证、权威背书
5. **A+页面与视觉化场景**：用图文讲故事、参数对比图、真实场景图

### 4. 广告与Listing联动
- 通过 SP Prompts 广告挖掘 20-30 个核心买家问题
- Listing 五点描述需要直接回应这些问题
- 支持竞价和A/B测试，并将广告反馈反哺 Listing 文案优化

### 5. 自检清单
- 标题是否 `<= 75` 字符
- 亮点是否 `<= 125` 字符
- 品牌名是否在前
- 五点是否回答了至少 5 个买家问题
- 是否包含场景词与信任背书
- 广告中的高频问题是否在 Listing 中被回答

## 对当前项目可能有影响的方向
- 需要把标题生成从“单段长标题”升级为“基础信息层 + 亮点层”的结构化输出
- 需要把 Rufus / COSMO / ABA / 场景词 / 买家问题引入 Listing 生成的中间结构，而不仅是最终纯文本
- 需要在前端增加规则校验和人工确认流程，例如标题长度、亮点长度、问题覆盖度、信任背书覆盖度
- 需要让广告词、搜索词、Q&A、评论痛点与 Listing 模块形成闭环
